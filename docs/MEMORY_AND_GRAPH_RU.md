# 🧠 Тройная гибридная память

Стандартные ИИ-агенты используют только Vector DB (RAG). Это приводит к "амнезии": при поиске по векторам теряются логические связи между объектами, а контекст забивается семантическим шумом. AAF решает это комбинацией трех баз данных.

## 1. SQL DB (PostgreSQL) - Картина мира
Хранит фундаментальные сущности (Mental State), долгосрочные задачи (Long-Term Tasks) и логи действий.
Здесь находится то, что агент **не имеет права забыть**. Сущности делятся на тиры (Critical, High, Medium, Low). Критические сущности всегда находятся в "Активном фокусе" агента (загружаются в системный промпт).

## 2. Vector DB (ChromaDB) - Семантика и Дневник
Используется для хранения сырых фактов, текстов статей и интроспекции.
Разделена на коллекции:
* `user_vector_db` - факты о пользователе.
* `agent_vector_db` - база знаний агента.
* `agent_thoughts_vector_db` - личный дневник. В цикле интроспекции агент записывает сюда свои глобальные выводы.

## 3. Graph DB (KuzuDB) - GraphRAG и Интуиция
Граф хранит логические связи (например: `[PicoClaw] - (СВЯЗАН С) -> [OpenClaw Lab]`).
AAF использует концепцию **"Интуиции 2-го уровня" (Depth 2)**. 
Когда вы спрашиваете агента о `PicoClaw`, `ContextBuilder`:
1. Находит в графе `PicoClaw`.
2. Смотрит прямые связи (Depth 1).
3. Смотрит косвенные ассоциации (Depth 2), игнорируя суперузлы (имя агента и админа), чтобы не было взрыва ассоциаций.
4. Собирает все найденные соседние узлы и делает точечный залп по Vector DB. 
В результате LLM получает феноменально точный контекст, понимая не только сам объект, но и то, с чем он связан.
